🔍 Motor de Visión e Ingesta (OCR)

Pipeline de procesamiento de imágenes para dos casos de uso: validación de recetas médicas (solicitante) y troqueles de medicamentos (donante).

// Pipeline unificado para receta y troquel
> OpenCV: Grayscale + Bilateral Filter (reducción de ruido)
> Tesseract Engine: LSTM (Deep Learning) OCR
> Parser Regex:
  - Receta → [Medicamento] [Dosis] [Médico] [Fecha]
  - Troquel → [Lote] [Vencimiento] [Droga] [Laboratorio]
// Output: JSON estructurado → trigger de consulta geoespacial

🔄 Flujos de Matching P2P + CAP

Dos flujos espejo que se complementan. El sistema mantiene oferta y demanda activa en tiempo real.

💊 Flujo A — Donante
1
Escaneo OCR del troquel

Valida lote, vencimiento y droga del medicamento excedente.

2
Consulta demanda cercana

Busca solicitudes activas del mismo medicamento en radio de 5km.

3
Match o espera activa

Si hay solicitante → notificación instantánea. Si no → queda en donantes_geo.

🏥 Flujo B — Solicitante
1
Escaneo OCR de receta

Valida autenticidad: médico, medicamento, dosis y fecha de emisión.

2
Consulta con prioridad

1° CAP zonal más cercano → 2° donante P2P cercano.

3
Match o espera activa

Si hay oferta → notificación con origen (CAP o donante). Si no → queda en solicitudes_geo.

🌍 Inteligencia Geográfica (PostGIS)

Tres tablas geoespaciales. La consulta de matching respeta el orden de prioridad: CAP zonal → donante P2P. Todo el proceso es gratuito para el usuario final.

-- Flujo A: Donante busca demanda activa cercana
SELECT solicitante_id, medicamento, ubicacion
FROM solicitudes_geo
WHERE medicamento = $1
  AND ST_DWithin(geom, $ubicacion_donante, 5000)
  AND activo = true;

-- Flujo B: Solicitante busca oferta con prioridad CAP → P2P
SELECT 'CAP' AS fuente, cap_id AS origen_id,
  medicamento, unidades_disponibles
FROM red_caps_geo
WHERE medicamento = $1
  AND ST_DWithin(geom, $ubicacion_solicitante, 5000)
  AND unidades_disponibles > 0

UNION ALL

SELECT 'DONANTE' AS fuente, donante_id AS origen_id,
  medicamento, cantidad_disponible
FROM donantes_geo
WHERE medicamento = $1
  AND ST_DWithin(geom, $ubicacion_solicitante, 5000)
  AND activo = true

-- CAP aparece siempre antes que DONANTE
ORDER BY fuente ASC;

📊 El Dato Estratégico — Inteligencia Sanitaria

La plataforma es 100% gratuita para donantes y solicitantes. Cada redistribución genera datos estructurados y geolocalizados que tienen valor para organizaciones que necesitan entender el sistema de salud.

Laboratorios Medicamento más donado por zona

Señal de sobreproducción regional. Permite ajustar volúmenes de fabricación y distribución.

Estado Medicamento más solicitado sin cobertura CAP

Mapea en tiempo real las fallas de distribución del sistema público de salud.

Estado Zonas con alta demanda sin CAP cercano

Informa decisiones de apertura de nuevos centros de atención primaria de salud.

Farmacéuticas Brechas de acceso por droga y región

Identifica poblaciones sin acceso real a medicamentos que existen en el mercado.

🏗️ Stack de Escalabilidad Industrial

Arquitectura modular para despliegue federal post-inversión.

⚙️
Core — Node.js Microservicios

Módulos desacoplados: OCR, matching geoespacial, notificaciones y analytics escalan de forma independiente.

🐳
Orquestación — Docker & Kubernetes

Nodos regionales para baja latencia. Escalado horizontal automático ante picos de publicaciones.

⚡
Caché — Redis

Gestión de solicitudes de donación activas y estados de matching en tiempo real. Reduce carga sobre PostgreSQL en consultas frecuentes.

🔒
Seguridad — AES-256

Encriptación de datos sensibles del paciente en reposo. Cumplimiento con normativa de protección de datos sanitarios.
